🧪 OQSE Rich Text Input
Three-tier security demo. Type any text below, then try the render buttons. Case A (Tier 1, no HTML) will throw a validation error if you include raw HTML tags. Case B (Tier 2 unsafe) executes payloads — click "Inject XSS" first to see the alert() fire. Case C (Tier 2 + DOMPurify) sanitizes the same input safely.
A · Pure Markdown (Tier 1) No HTML allowed
sdk.text.renderHtml(raw, { requirements: { features: [] } })
Throws a validation error if raw HTML tags are detected.
B · HTML Enabled (Tier 2) — Unsafe ⚠ XSS risk
sdk.text.renderHtml(raw, { requirements: { features: ['html'] } })
No sanitizer — injected scripts & event handlers execute.
C · HTML Enabled (Tier 2) — Safe ✅ DOMPurify
sdk.text.renderHtml(raw, { requirements: { features: ['html'] }, sanitizer: DOMPurify.sanitize })
DOMPurify strips dangerous attributes before DOM insertion.
🔬 Tokenized — sdk.text.parseTokens(raw)
Each token is individually escaped — always safe regardless of tier.
